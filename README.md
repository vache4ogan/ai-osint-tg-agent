
# 🤖 Автономный мультиагентный OSINT Telegram-бот (LangGraph + CRAG)

Автономная мультиагентная система, предназначенная для мониторинга IT-индустрии, агрегации технологического контента и автоматической генерации фактчекинговых публикаций для Telegram-каналов.

Проект построен на базе фреймворка **LangGraph** для оркестрации команды специализированных ИИ-агентов с использованием локальных LLM и паттерна **Corrective RAG (CRAG)** для полной ликвидации галлюцинаций.

---

## 📐 Архитектура системы и воркфлоу агентов

В проекте реализован продвинутый пайплайн на основе **Рефлексии (Reflection)** и **Адаптивного поиска (CRAG)**:

1. **Scout (Парсер):** Мониторит RSS-ленты и парсит сырой HTML с технологических платформ (например, Хабр). Кэширует обработанные URL в базе данных SQLite для обеспечения идемпотентности (защиты от дубликатов).
2. **Retriever Node:** Выполняет семантический поиск по локальной векторной базе данных (ChromaDB) с использованием мультиязычных эмбеддингов.
3. **Router (CRAG):** Оценивает качество извлеченного контекста. Если в локальной базе знаний недостаточно информации для раскрытия темы, он динамически активирует инструмент **Web Search** (API DuckDuckGo) для добора актуальных фактов из интернета.
4. **Writer Node:** Генерирует черновик публикации, опираясь *строго* на собранные факты, соблюдая заданную SMM-структуру и tone of voice.
5. **Critic Node (Валидация через Pydantic):** Выступает в роли безжалостного главного редактора. Он проверяет черновик на соответствие фактам, используя жесткие схемы валидации **Pydantic**. Если обнаружены несостыковки или галлюцинации, он возвращает структурированный фидбек Писателю на доработку (максимум 3 итерации).
6. **Human-in-the-Loop:** После того как Критик верифицирует текст, бот отправляет финальный вариант в Telegram-панель администратора на модерацию (кнопки: Опубликовать / Перегенерировать / Отклонить).

## 🛠️ Технологический стек

* **Оркестрация агентов:** LangGraph, LangChain, Ollama (`llama3` / `qwen2.5`)
* **Векторная база и эмбеддинги:** ChromaDB, HuggingFace Embeddings (`paraphrase-multilingual-MiniLM-L12-v2`)
* **Данные и инфраструктура:** SQLite (кэширование запросов), BeautifulSoup4, Feedparser
* **Интерфейс и асинхронность:** Aiogram 3.x (Асинхронный фреймворк для Telegram-ботов)

## 🚀 Быстрый запуск


### 1. Клонирование репозитория и настройка окружения

git clone https://github.com/ваш-никнейм/langgraph-rag-content-bot.git

cd langgraph-rag-content-bot


# Создание и активация виртуального окружения

python3.10 -m venv venv

source venv/bin/activate


# Установка зависимостей

pip install -r requirements.txt


### 2. Запуск локальной LLM через Ollama

Убедитесь, что Ollama установлена и запущена на вашем компьютере:

ollama run llama3:latest


### 3. Настройка конфигурации

Создайте файл src/config.py в директории проекта:

# src/config.py

token = "ВАШ_ТОКЕН_ТЕЛЕГРАМ_БОТА"


### 4. Запуск приложения

Запустите Telegram-бот как модуль пакета из корневой директории:

python -m src.tg_bot
